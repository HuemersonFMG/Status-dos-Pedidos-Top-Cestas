Sistema de Rastreamento de Pedidos e Entregas

📘 Visão Geral do Sistema

O sistema de rastreamento da Top Alimentos foi desenvolvido para permitir o acompanhamento dos pedidos realizados pelos clientes, exibindo o status atualizado da entrega diretamente a partir do banco de dados do ERP Sankhya.

O sistema possui três módulos principais:

  • Gerador de Links: utilizado pela equipe interna para gerar links de rastreamento.
  • Página do Cliente: onde o cliente acompanha o status do pedido.
  • Página do Vendedor - Específico PAP: permite visualizar diversos pedidos em lote a partir do CNPJ e Período de entrega.
Fluxo Geral do Sistema:

ERP Sankhya → API Node.js → Views do Banco → Geração de Links → Cliente acompanha entrega em tempo real

🔗 Módulo Gerador de Links

O arquivo gerador.html é utilizado pela equipe da Top Alimentos para gerar links individuais ou em lote para os clientes.

📌 Objetivo

Criar links seguros de rastreamento dos pedidos.

📋 Filtros Disponíveis

Filtro Descrição
CPF/CNPJ Busca todos os pedidos de um cliente em um período de 60 dias.
Ordem de Carga Busca pedidos vinculados a uma carga (Minuta de carregamento) específica.
Pedidos (Número único) Busca pedidos específicos separados por vírgula.
Data (Liberação) Busca pedidos pela data de liberação do pedido via PCP.

⚙️ Como Utilizar

  • Abra o módulo Gerador de Links.
  • Escolha apenas um filtro.
  • Clique em: Gerar Links Clientes.
  • O sistema exibirá os links encontrados.
  • Utilize os botões:
    • 📋 Copiar - Copia um link para a área de transferenica.
    • 🔗 Abrir - Abre a página referente ao Link exibindo as informações do Pedido.
    • 📲 WhatsApp - Abre o WhatsApp Web para colar o link e enviar ao cliente.

Na barra de títulos temos dois botões de ação.

  • 🔝 Baixar Todos Comprovantes: Com este botão podemos baixar todos os comprovantes da lista de links gerados, caso existam, e salvar em uma pasta local.
  • 📘 Manual: Com este botão podemos visualizar um Mnaual do usuário.
O sistema bloqueia o uso simultâneo de múltiplos filtros, evitando buscas incorretas ou excessivas.

📦 Página de Rastreamento do Cliente

A página index.html é acessada diretamente pelo cliente através do link gerado.

📌 Informações Exibidas

  • Número do Pedido
  • Número da Nota
  • Cliente
  • CPF/CNPJ
  • Data do Pedido
  • Ordem de Carga
  • Transportadora
  • Status Atual da Entrega
  • Comprovante da Entrega

📷 Comprovantes

Quando disponível, o sistema poderá exibir:

  • 📷 Foto da entrega - Gerado a partir do aplicativo de entregas (Motorists da Top Cestas)
  • 📄 PDF do comprovante - Gerado via API de uma transportadora contratada (Solistica)
Todos os dados exibidos são consultados em tempo real diretamente do banco de dados do Sankhya.

👨‍💼 Página do Vendedor - este móduo ainda está sendo desenvolvido.

O módulo vendedor permite ao vendedor acompanhar diversos pedidos simultaneamente a partir de um ou mais CNPJs informados. Este módulo está sendo projetado para gerar links de acompanhamento de entrega de pedidos PAP, facilitando a administração e download de comprovantes de entrega.

📌 Objetivo

Gerar um único link contendo vários pedidos filtrados por:

  • CPF/CNPJ - pode filtrar por CNPJ ou CPF.
  • Período de campanha - Período de entregas (PAP)

⚙️ Como Funciona

  • O gerador cria um token seguro.
  • O vendedor recebe um único link.
  • Ao abrir o link:
    • Todos os pedidos são carregados
    • Os status são exibidos em cards
    • O vendedor pode abrir o pedido
    • Pode enviar pelo WhatsApp
    • Pode exibir a foto dos comprovantes
Este recurso é ideal para pedidos gerados por (Multi Entregas) no Planejamento de Entregas e é mais específoco para entregas de modalidade Porta a Porta (PAP ), gerenciando e reduzindo o trabalho operacional.

🔐 Segurança e Controle de Acesso

✅ Pontos Fortes

  • Uso de tokens SHA-256 para proteção dos links.
  • Validação de token em todas as rotas críticas.
  • Cache de autenticação no Sankhya.
  • Separação entre API e Front-End.
  • Controle de filtros no front-end.
  • Consulta apenas através de Views do banco.
  • Não expõe diretamente o banco ao usuário final.

⚠️ Pontos Fracos Identificados

  • SECRET está fixo no backend.
  • Usuário e senha do Sankhya estão expostos no backend.
  • Não existe expiração dos tokens.
  • Não existe controle de tentativas de acesso.
  • Não há autenticação administrativa no gerador.
  • Bloqueio de F12 não impede acesso técnico real.
  • Algumas rotas ainda permitem parâmetros diretos via URL, mas já exite uma camada de proteção (✅ token SHA-256).

🚀 Sugestões de Melhoria

  • Mover SECRET, USER e PASS para variáveis de ambiente.
  • Implementar JWT com expiração.
  • Adicionar autenticação administrativa.
  • Criar logs de auditoria.
  • Adicionar Rate Limit nas rotas.
  • Criar expiração automática dos links.
  • Validar origem das requisições.
  • Implementar CDN para comprovantes.
  • Melhorar proteção contra download massivo.

✅ Conclusão

O sistema de rastreamento da Top Alimentos fornece uma solução moderna e eficiente para acompanhamento de pedidos em tempo real utilizando integração direta com o ERP Sankhya.

A arquitetura atual atende muito bem o processo operacional, principalmente na automação da geração de links, acompanhamento de entregas e visualização de comprovantes.

Pode ser implementdo uma melhoria no sistema segurança da API, onde o sistema poderá alcançar um nível ainda maior de confiabilidade e escalabilidade.

  • Nome do cliente
  • CPF/CNPJ
  • Número do pedido
  • Dados de entrega
  • Status logístico
  • Comprovante de entrega
  • Foto da entrega
  • Possível assinatura
  • Dados comerciais
  • Histórico operacional

    Pela LGPD:

  • ✅ CPF = dado pessoal
  • ✅ foto de entrega = dado pessoal
  • ✅ comprovante assinado = dado pessoal
  • ✅ nome + pedido = dado pessoal

    O que está alinhado com a LGPD neste sistema:

  • 1. Não expõe o banco diretamente

    • Não abriu acesso direto ao Oracle
    • Não expôs query livre
    • Não liberou conexão externa

      • ✅ Muito bom para LGPD e segurança.

  • 2. Uso de Views

    • limita os dados
    • controla exposição
    • reduz vazamento

      • ✅ Boa prática LGPD.

  • 3. Uso de tokens

    • Isso reduz acesso indevido simples.

      • ✅ Boa prática LGPD.

  • 4. Separação de módulos

    • cliente
    • vendedor
    • backend

      • ✅ Isso ajuda no princípio do menor privilégio.

      ✅ Boa prática LGPD.

  • 5. Não coleta dados desnecessários

      • A API aparentemente usa apenas:

    • dados operacionais
    • dados logísticos

      • Evitando:

    • biometria
    • localização GPS
    • dados sensíveis

      • ✅ Excelente.

      ✅ Boa prática LGPD.

    ⚠️ Pontos de atenção LGPD

  • 1. Links podem ser compartilhados
  • 2. CPF completo exposto
  • 3. Comprovantes contendo assinatura e dados pessoais
      • LGPD exige
    • ✅ finalidade clara
    • ✅ proteção adequada
    • ✅ retenção controlada
    • ✅ acesso restrito
  • 5. Falta de auditoria
    • quem gerou links
    • quem acessou comprovantes
    • horário
    • IP
    • volume
  • 6. SECRET e credenciais no código
  • ❌ Ausência de política de retenção - (Expiração dos links)
  • ❌ Sem expiração de token
  • ❌ Falta de consentimento/transparência
