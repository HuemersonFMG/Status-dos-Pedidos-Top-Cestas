📘 Visão Geral do Sistema

O sistema de rastreamento da Top Alimentos foi desenvolvido para permitir o acompanhamento dos pedidos realizados pelos clientes, exibindo o status atualizado da entrega diretamente a partir do ERP Sankhya.

O sistema possui quatro módulos principais:

  • Login Administrativo: valida usuário e senha diretamente no Sankhya antes de liberar os geradores.
  • Gerador de Links: Utilizado pela equipe interna para gerar links de rastreamento de pedidos de entrega direta no cliente, por: CPF/CNPJ, ordem de carga, pedidos, NF-e ou data de produção. Neste caso gerar via CNPJ_CPF é indicado para clientes Simples descarga.
  • Gerador Comercial: utilizado para gerar links de pedidos Porta a Porta onde o volume de remessas é muito alto e o que destaca neste processo é a informação de (Período de Entregas), possuindo os filtros combinados por CPF/CNPJ e Período PAP.
  • Página do Cliente: onde o cliente acompanha o status do pedido sem precisar de login.
Fluxo Geral do Sistema:

Usuário interno → Login Sankhya → Gerador de Links → API Node.js → Views Sankhya → Geração de link seguro → Cliente acompanha entrega

🔐 Login Administrativo

Antes de acessar o gerador.html ou o geradorcomercial.html, o usuário interno deve informar usuário e senha do Sankhya.

📌 Objetivo

Impedir que pessoas não autorizadas acessem os geradores de links e façam consultas administrativas.

⚙️ Como Funciona

  • O usuário acessa o gerador.
  • Se ainda não estiver autenticado, o sistema redireciona para login.html.
  • O usuário informa login e senha do Sankhya.
  • A API valida as credenciais usando autenticação no Sankhya.
  • Se o login estiver correto, o sistema libera o acesso ao gerador.
  • Se a senha estiver incorreta, o sistema permanece na tela de login.
  • Após várias tentativas inválidas, o login pode ficar bloqueado temporariamente.
  • Sobre os Status Possíveis para os pedidos

Importante: o cliente final que acessa o index.html não precisa de usuário e senha do Sankhya. O acesso do cliente continua sendo feito exclusivamente pelo link com token seguro.

🔗 Módulo Gerador de Links

O arquivo gerador.html é utilizado pela equipe da Top Alimentos para gerar links individuais ou em lote para os clientes.

📌 Objetivo

Criar links seguros de rastreamento dos pedidos.

📋 Filtros Disponíveis

Filtro Descrição
CPF/CNPJ Busca todos os pedidos de um cliente de Entrega Direta dentro do período definido na restrição do Sistema (≈60 dias). É importante não gerar links de entregas em residência - PAP nesta tela para não sobrecarregar o sistema, para PAP utilize o Gerador Comercial.
Ordem de Carga Busca pedidos vinculados a uma Ordem de Carga específica Minuta de Carregamento. Esta consulta pode ser muito útil por exemplo no caso de pedidos da Pasi.
Pedidos / ÚNICO Busca por um ou vários números de pedidos específicos separados por vírgula.
Número da NF-e Busca um ou vários números de notas fiscais específicas separadas por vírgula.
Data Busca pedidos pela data de liberação dos pedidos Fluxo PCP. Seria como dizer que está buscando pela data de Produção, neste caso serão listados tanto entregas diretas no Cliente como as entregas em residência.

É importante observar que o rastreamento de um pedido deve iniciar a partir da informação de que o produto foi produzido (Cesta, Kit Higine ou outros), na Top Cestas identificamos esta data como sendo a data de liberação dos Pedidos no PCP, o que gera uma Nota Temporária.

Ainda está em análise o processo de automação, ou seja, a intenção é que os links de rastreio sejam enviados ao cliente de forma totalmente automática, e esta automação dever ser previsata no fluxo. Desta forma para Entregas Diretas e Auto Distribuição seria a partir do momento em que o pedido é confirmado gerando a Nota de Venda (Temporária). Para as Multi Entregas e Porta a Porta este link deve ser gerdo assim que o reultado do Planejamento de entregas for cofirmado, ou seja na confirmação das remessas.

Regras sugeridas:

    O ponto correto para disparar é quando o documento já representa entrega real.

  • ED / RT / AD: Gerar link quando o fluxo transformar pedido de venda em nota de venda com TOP onde AD_FUSION = 'S'.
  • ME: Gerar link para as notas/remessas que aparecem na rota, não para a nota de cobrança se ela não sobe para rota.
  • PAP: Gerar link para as remessas/pedidos TOP 1004 quando AD_FUSION = 'S' e vinculadas aos contatos participantes do planejamento.
  • Refrenicas de ontatos: Usar TGFCAB.CODCONTATO + TGFCAB.CODPARC para buscar TGFCTT, validando se é o contato correto/participante do planejamento.

A API Node não deve “adivinhar ” tudo varrendo 30.000 registros sem controle. O Sankhya ou uma rotina agendada deve apenas registrar os documentos elegíveis na fila. Depois o Node processa em lote, gera os links e envia e-mail/WhatsApp de acordo com o cadastro de contatos dos parceiros.

Ponto importante: não colocar envio de WhatsApp/e-mail dentro da confirmação/faturamento. Isso pode travar o fluxo. O correto seria Gravar fila e processar posteriormente.

⚙️ Como Utilizar - Gerar links de pedidos

  • Abra o módulo Gerador de Links.
  • Informe usuário e senha do Sankhya, caso solicitado.
  • Escolha apenas um filtro.
  • Clique em Gerar Links.
  • O sistema exibirá os links encontrados.
  • Utilize os botões disponíveis em cada link:
  • Ao finalizar utilize o botão para Sair
  • Se o sistama parar por ociosidade utilize o botão Recarregar
    • 📋 Copiar - Copia o link para a área de transferência.
    • 🔗 Abrir - Abre a página de rastreamento do pedido.
    • 📲 WhatsApp - Abre o WhatsApp Web para envio do link ao cliente.

Na barra superior existem botões de ação:

  • 🔝 Baixar Todos Comprovantes: baixa todos os comprovantes disponíveis da lista gerada em um único arquivo ZIP.
  • 📘 Manual: abre este manual do usuário.
  • 👨‍💼 Gerador Comercial: abre o módulo comercial para geração por CPF/CNPJ + Período PAP.
Para a tela do Gerador principal, o sistema bloqueia o uso simultâneo de múltiplos filtros, evitando buscas incorretas ou excessivas.

👨‍💼 Gerador Comercial

A tela Gerador Comercial é utilizada para geração de links em lote com foco em operações comerciais, campanhas e principalmente entregas PAP, onde o pedido deve conter além do CNPJ/CPF a informação de Período de Entrega.

📌 Objetivo

Permitir a geração de links usando filtros combinados obrigatórios limitando a busca em um período de entrega específico que normalmente é utilizado nas entregas em residência (PAP), o que evita gerar links em quantidades muito excessivas e travamento do sistema. Com foco principal em atender uma demanda de envio de comprovantes em lote para os clientes quando da solicitação destes pelo cliente.

📋 Filtros Obrigatórios

Filtro Descrição
CPF/CNPJ Documento do cliente parceiro.
Período PAP Campo obrigatório da Tela de Pedidos, utilizado para identificar o período de campanha ou entrega PAP.

⚙️ Como Utilizar

  • Acesse o Gerador Comercial pelo botão no Gerador principal.
  • Informe CPF/CNPJ.
  • Informe o Período PAP exatamente conforme cadastrado no Sankhya.
  • Clique em Gerar Links.
  • O sistema retornará somente pedidos que atendam aos dois filtros combinados.

Para consultar o CNPJ e Período corretos para a utilização correta nesta tela, pode ser utilizado o Relatório de Comprovantes gerados ou a partir de uma consulta direta no Portal de vendas a partir do numero de um dos pedidos.

No Gerador Comercial, CPF/CNPJ e Período PAP devem ser usados juntos. Se um dos campos não for informado, o sistema solicita o preenchimento antes da consulta.

📦 Página de Rastreamento do Cliente

A página Status do Pedido cujo link foi devidamente gerado, é acessada diretamente pelo cliente para o qual o link foi enviado, no momento este envio será feito manualmente mas já estamos trabalhando em um processo de automação. Por segurança o link possui o numero do Pedido e uma Hash SHA-256 do pedido que valida acesso ao pedido, protege o link e impede acesso direto apenas pelo número do pedido evitando que um cliente consiga abrir o rastreador de outro cliente.

📌 Informações Exibidas

  • Número do Pedido - Dados não Pessoais.
  • Número da Nota - Dados não Pessoais.
  • Cliente - Dados não Pessoais.
  • CPF/CNPJ - Dados Pessoais quando CPF.
  • Data do Pedido - Dados não Pessoais.
  • Ordem de Carga - Dados não Pessoais.
  • Transportadora - Dados não Pessoais.
  • Status Atual da Entrega - Dados não Pessoais.
  • Comprovante da Entrega - Quando Canhoto da NFe ou CTE da Transpotadora não possui dados Pessoais, mas no Comprovante PAP já teremos Dados Pessoais.

📍 Status Possíveis

Aguardando rota🧺 Sua cesta foi produzida e aguarda transporte..

Emtregamdo🚚 Sua Cesta foi entregue a Transportadora.

Nota Emitida🧾 Sua Nota Fiscal foi emitida e o pedido está pronto para entrega.

Entregue✅ Pedido entregue com sucesso.

📷 Comprovantes

Quando disponível, o sistema poderá exibir:

  • 📷 Foto da entrega - Gerada a partir do aplicativo de entregas.
  • 📄 PDF do comprovante - Gerado via transportadora ou integração externa.
O cliente final acessa somente pelo link com token. Não é necessário login Sankhya para consultar o pedido.

📁 Download de Comprovantes em ZIP

O sistema atual permite baixar todos os comprovantes disponíveis da lista gerada em um único arquivo ZIP.

📌 Como Funciona

  • O usuário gera uma lista de links no Gerador principal ou no Gerador Comercial.
  • Clica em Baixar Todos Comprovantes.
  • A API busca os comprovantes disponíveis no Sankhya.
  • O sistema prioriza imagem quando existir foto da entrega.
  • Se não houver imagem, tenta localizar PDF/base64 do comprovante.
  • Todos os arquivos encontrados são compactados em um único ZIP.

📝 Nome dos Arquivos

Os comprovantes são salvos com a nomenclatura:

Comprovante_NUNOTA_NOMEPARC.pdf
Comprovante_NUNOTA_NOMEPARC.jpg

Exemplo:

Comprovante_543012_JOAO_DA_SILVA.pdf
Comprovante_543013_MARIA_APARECIDA.jpg
Essa nomenclatura facilita conferência, auditoria, envio por e-mail, organização local e identificação do cliente sem precisar abrir o arquivo.

🔐 Segurança e Controle de Acesso || Para avaliação e suporte TI.

✅ Pontos Fortes

  • Uso de tokens SHA-256 para proteção dos links dos clientes.
  • Validação de token nas rotas públicas de pedido e comprovante.
  • Login administrativo validado contra usuário e senha do Sankhya.
  • Controle de tentativas inválidas no login administrativo.
  • Sessão administrativa para acesso aos geradores.
  • Separação entre acesso administrativo e acesso do cliente final.
  • Consulta de dados através de Views.
  • Não expõe conexão direta com o banco ao usuário final.
  • Download de comprovantes concentrado em rota protegida.

⚠️ Pontos de Atenção

  • SECRET ainda deve preferencialmente ser movido para variável de ambiente.
  • Usuário técnico e senha técnica do Sankhya devem preferencialmente ficar em variáveis de ambiente.
  • Os links públicos ainda não possuem expiração automática.
  • Links podem ser compartilhados por terceiros.
  • Bloqueio de F12 não impede acesso técnico real.
  • Comprovantes podem conter dados pessoais e devem ser tratados com cuidado.

🚀 Sugestões de Melhoria

  • Mover SECRET, USER e PASS para variáveis de ambiente.
  • Implementar expiração automática dos links públicos.
  • Criar logs de auditoria para geração de links e download de comprovantes.
  • Registrar usuário administrativo que gerou links ou baixou comprovantes.
  • Adicionar Rate Limit nas rotas sensíveis.
  • Validar origem das requisições.
  • Criar política de retenção para comprovantes.
  • Melhorar proteção contra download massivo.

Pontos alinhados com a LGPD

    A aplicação manipula:

  • Nome do cliente
  • CPF/CNPJ
  • Número do pedido
  • Número da nota fiscal
  • Dados de entrega
  • Status logístico
  • Comprovante de entrega
  • Foto da entrega
  • Possível assinatura
  • Dados comerciais
  • Histórico operacional

    Pela LGPD:

  • ✅ CPF = dado pessoal
  • ✅ Foto de comprovante de entrega = pode possuir dado pessoal
  • ✅ comprovante assinado = dado pessoal
  • ✅ nome + pedido ao baixar comprovantes = dado pessoal

    O que está alinhado com a LGPD neste sistema:

  • 1. Não expõe o banco diretamente

    • Não abriu acesso direto ao Oracle
    • Não expôs query livre ao usuário final
    • Não liberou conexão externa com o banco

      • ✅ Muito bom para LGPD e segurança.

  • 2. Uso de Views

    • Limita os dados retornados
    • Controla a exposição
    • Reduz risco de vazamento

      • ✅ Boa prática LGPD.

  • 3. Uso de tokens

    • Reduz acesso indevido simples.

      • ✅ Boa prática LGPD.

  • 4. Login administrativo

    • Restringe acesso aos geradores.
    • Exige usuário válido no Sankhya.
    • Reduz risco de geração indevida de links.

      • ✅ Melhoria importante de segurança.

  • 5. Separação de módulos

    • Cliente
    • Gerador interno
    • Gerador comercial
    • Backend

      • ✅ Isso ajuda no princípio do menor privilégio.

  • 6. Não coleta dados desnecessários

      • A API usa principalmente:

    • dados operacionais
    • dados logísticos
    • dados comerciais necessários ao rastreamento

      • E evita:

    • biometria
    • localização GPS contínua
    • dados sensíveis desnecessários

      • ✅ Boa prática LGPD.

    ⚠️ Pontos de atenção LGPD

  • Links podem ser compartilhados.
  • CPF/CNPJ pode ser exibido ao cliente.
  • Comprovantes podem conter assinatura e dados pessoais.
  • Arquivos ZIP devem ser armazenados e compartilhados com cuidado.
  • É recomendável auditoria de geração de links e downloads.
  • É recomendável política de retenção e expiração dos links.
  • É recomendável manter credenciais e SECRET fora do código-fonte.

    • 🛡️ O sistema está “ilegal”?

    Não necessariamente.

    A LGPD não exige perfeição absoluta.

    Ela exige:

  • ✅ finalidade legítima
  • ✅ necessidade
  • ✅ segurança razoável
  • ✅ controle operacional
  • ✅ boa-fé

✅ Conclusão

O sistema de rastreamento da Top Alimentos fornece uma solução moderna e eficiente para acompanhamento de pedidos em tempo real utilizando integração direta com o ERP Sankhya.

A arquitetura atual atende o processo operacional de geração de links, acompanhamento de entregas, visualização de comprovantes e download em lote.

As melhorias recentes aumentaram a segurança e a usabilidade do projeto, principalmente com login administrativo Sankhya, Gerador Comercial, download ZIP e nomenclatura dos comprovantes com pedido e parceiro.
