Módulo de Etiquetas de Cestas acabadas - Implementação via API

Mapeamento de dados Sankhya para geração de etiquetas de produção e separação

1. Objetivo

Este documento descreve o mapeamento técnico necessário para criação do módulo web de geração de etiquetas de cestas, substituindo a aplicação desktop atual e eliminando a necessidade de servidor local.

Resumo: Gerar etiquetas de controle de produção e separação a partir do número único do pedido, buscando os dados diretamente no Sankhya ERP, eliminando ao máximo a entrada de dados manualmente e eliminando a aplicação desktop com servidor local. Com a finalidade de implementação via API para um servidor em nuvem que atenda a demanda para todas as unidades sem precisar intalação direto nas máqunas locais, melhorando o processo de manutenção e atualizações.

3. Regra de geração das linhas

O sistema deve gerar automaticamente as linhas de etiquetas com base no total de cestas e na quantidade de cestas por palete.

TOTAL_PALETES = CEIL(TOTAL_CESTAS / CESTAS_POR_PALETE) Exemplo: TOTAL_CESTAS = 60 CESTAS_POR_PALETE = 29 TOTAL_PALETES = 3 Linha 1 = 29 cestas Linha 2 = 29 cestas Linha 3 = 2 cestas

6. View sugerida

Recomenda-se criar uma View leve, consultada sempre por NUNOTA.

CREATE OR REPLACE VIEW VW_ETIQUETAS_CESTAS AS SELECT 1 AS DON, CAB.NUNOTA, CAB.NUMNOTA, CAB.CODPARC, PAR.NOMEPARC, I.CODPROD AS CODPROD_COMPOSICAO, P.DESCRPROD AS COMPOSICAO, CAB.QTDVOL AS TOTAL_CESTAS, CAB.PESOBRUTO AS PESO, CAB.VLRNOTA AS VALOR, CAB.DTNEG, CAB.CODTIPOPER, CAB.DHTIPOPER FROM TGFCAB CAB INNER JOIN TGFPAR PAR ON PAR.CODPARC = CAB.CODPARC LEFT JOIN TGFITE I ON I.NUNOTA = CAB.NUNOTA AND I.USOPROD = 'V' LEFT JOIN TGFPRO P ON P.CODPROD = I.CODPROD WHERE 1 = 1 AND CAB.QTDVOL > 0;

8. Recomendação de implementação

A primeira versão deve apenas consultar o Sankhya, gerar as linhas e imprimir as etiquetas. Não é recomendado gravar ou alterar dados fiscais, financeiros, estoque ou pedido nesta fase.
