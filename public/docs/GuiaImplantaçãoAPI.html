API de Monitoramento de Pedidos - Windows Server 2022

📌 Objetivo

Implantar a aplicação Node.js de monitoramento de pedidos em ambiente de produção com estabilidade e segurança.

🧠 Arquitetura

Usuário (Browser) ↓ Node.js (Express) ↓ API Sankhya

⚙️ Pré-requisitos

  • Acesso administrativo ao servidor
  • Portas liberadas (ex: 5050)
  • Acesso ao Sankhya
  • Internet para instalação

📦 Instalar Node.js

Baixe em: https://nodejs.org

node -v npm -v

📁 Publicar Projeto

C:\apps\status-pedidos

Copiar arquivos:

server.js package.json /public

📥 Instalar Dependências

cd C:\apps\status-pedidos npm install

🔐 Variáveis de Ambiente

SECRET=chave_super_secreta USER=usuario PASS=senha PORT=5050

🔄 Executar com PM2

npm install -g pm2 pm2 start server.js --name status-pedidos pm2 save pm2 startup

🔍 Monitoramento

pm2 list pm2 logs pm2 restart status-pedidos

🌐 Acesso

http://IP_DO_SERVIDOR:5050

🧪 Testes

http://IP:5050/gerador.html http://IP:5050/index.html?nunota=XXXX&token=XXXX

🔐 Segurança

  • Não deixar credenciais no código
  • Usar .env
  • Monitorar logs

📦 Estrutura

C:\apps\status-pedidos │ ├── server.js ├── package.json ├── .env └── public

🚨 Problemas Comuns

Porta não responde:
Verifique PM2 e firewall
Erro 500: pm2 logs
Erro Sankhya:
Validar credenciais e URL

🚀 Melhorias Futuras

  • HTTPS (SSL)
  • NGINX ou IIS
  • Docker
  • Monitoramento avançado

✅ Conclusão

Sistema pronto para produção, com execução contínua, integração com Sankhya e acesso via rede interna.
